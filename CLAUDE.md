# CLAUDE.md — このプロジェクトでの作業ルール

## README の更新ルール

コードに変更を加えたときは、**必ず README.md を合わせて更新してからコミットする**。

更新が必要な箇所：

- **アーキテクチャ図** — 構成・データフロー・使用技術に変化があった場合
- **デプロイシーケンス図** — デプロイ手順が変わった場合
- **技術スタック表** — ライブラリ・インフラの追加・変更があった場合
- **バージョン履歴** — 毎回必ず追記する（日付・バージョン・変更内容）

### バージョン番号のルール

| 変更の種類 | バージョン |
|-----------|-----------|
| 機能追加・構成変更 | v0.x → v0.(x+1) |
| バグ修正・小変更 | v0.x.y → v0.x.(y+1) |
| 認証導入など大きな節目 | v0.x → v1.0 |

## デプロイ手順

```bash
npm run build
git add .
git commit -m "..."
git push
npx firebase deploy --token "..."
```

トークン発行: `npx firebase login:ci`（プロジェクトディレクトリ内で実行）
